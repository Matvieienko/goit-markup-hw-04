## Критерії приймання роботи наставником

### Проєкт

- У корені проєкту є папка images з зображеннями
- У корені проєкту є папка css з файлами стилів
- Всі стилі написані у своїх відповідних файлах, які знаходиться в папці css та коректно підключені
- У назвах файлів відсутні кирилиця, пробіли. Назви містять тільки літери, числа і слова англійської мови
- Вихідний код відформатований за допомогою Prettier
- Усі зображення та текстовий контент взяті з макета
- Підключений нормалізатор стилів modern-normalize (мініфікована версія)
- Код написаний з дотриманням настанови

### Оформлення

- Допускається глобальне скидання стилів за селектором тегу для елементів `<h1>...<h6>`, `<p>` і `<ul>`
- Непотрібно використовувати тіні та інші декоративні ефекти (це предмет наступних модулів), працюйте з геометрією та позиціюванням
- В елементів відсутні зовнішні відступи (властивість `margin`) що «пробивають» батьківський елемент
- Для вертикальних відступів між двома сусідніми елементами використовується властивість `margin`
- Для проміжку між рамкою батька і його дитиною використовується властивість `padding`
- Розміри зовнішніх відступів (властивість `margin`) і внутрішніх полів (властивість `padding`) елементів задані точно за макетом
- Створений загальний допоміжний клас `.container` для центрування і обмеження контенту по ширині
- Ширина «контейнера» відповідає макету і дорівнює `1158px`
- «Контейнер» обгортає контент хедера, футера і секцій. Тобто знаходиться всередині них
- Для розташування елементів використовується Flexbox, але тільки там, де це необхідно. Наприклад у шапці, навігації, списках в секціях тощо, тобто там, де розташувати елементи горизонтально по-іншому неможливо
- Фінальні розміри блоків в браузері відповідають макету
- В елементів відсутня фіксована висота, вона визначається їх контентом
- Висота хедера встановленна за рахунок контенту та вертикальних паддінгів посилань
- У хедера є нижня рамка, необхідно сильно наблизити макет, щоб побачити її
- Висота усіх посилань в хедері визначається висотою контента та вертикальними падінгами
- Секції розташовані одна під одною як стопка книг, без зовнішніх відступів
- Для всіх секцій використовується один клас `.section`, якому задані верхні і нижні падінги по `120px`, що відсувають контент всередину секції
- Для правильного приховування заголовка другої секції в файлі в `common.css` використовується патерн – `sr-only`:

```css
.sr-only {
  position: absolute;
  white-space: nowrap;
  width: 1px;
  height: 1px;
  overflow: hidden;
  border: 0;
  padding: 0;
  clip: rect(0 0 0 0);
  clip-path: inset(100%);
  margin: -1px;
}
```

- У картках секції Our Portfolio є рамка (властивість `border`), але тільки у нижній частині картки
